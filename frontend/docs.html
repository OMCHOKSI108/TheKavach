API Documentation

Authentication

All endpoints except /generate-key and /health require an API key passed in the Authorization header.

Authorization: Bearer tk_your_api_key_here

POST /api/generate-key

Generate a new API key for accessing the telemetry API.

Request Body

{
  "name": "John Doe"
}

Response

{
  "api_key": "tk_a1b2c3d4e5f6...",
  "name": "John Doe",
  "created_at": "2026-05-19T00:00:00",
  "message": "API key generated successfully."
}

GET /api/logs

Fetch a batch of synthetic cybersecurity logs.

Query Parameters

ParameterTypeDefaultDescription
countint50Number of logs (1-500)
threat_labelstring-Filter: benign, suspicious, malicious
log_typestring-Filter: firewall, network, application, ids
protocolstring-Filter: TCP, UDP, HTTP, HTTPS, etc.

Response

{
  "count": 50,
  "logs": [
    {
      "id": "uuid",
      "timestamp": "2026-05-19T12:30:00",
      "source_ip": "192.168.1.42",
      "dest_ip": "192.168.1.100",
      "protocol": "TCP",
      "action": "blocked",
      "threat_label": "malicious",
      "log_type": "firewall",
      "bytes_transferred": 45123,
      "user_agent": "Nmap Scripting Engine",
      "request_path": "/admin/config"
    }
  ]
}

GET /api/stream

Server-Sent Events endpoint for real-time log streaming.

Query Parameters

ParameterTypeDefaultDescription
threat_labelstring-Filter by threat level
intervalfloat1.0Seconds between entries (0.1-10)

JavaScript Example

const evtSource = new EventSource('/api/stream?interval=0.5');
evtSource.onmessage = (event) => {
  const log = JSON.parse(event.data);
  console.log(log);
};
